# Idea: Interactive session model and launch command design

**Date:** 2026-03-14
**Status:** raw

---

## The Idea

Current 'work' and 'idea' commands launch separate sessions per persona. Alternative model: one session loads all personas, switch organically via natural language (e.g. 'as the Oracle, let's discuss X'). Either way, commands should follow 'domus launch <persona>' pattern. Must also distinguish interactive (human-present) vs background (worker) launches explicitly — these work very differently and need separate command paths.
