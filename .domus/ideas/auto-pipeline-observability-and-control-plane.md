**Captured:** 2026-03-15
**Status:** raw

## The Idea

How does the human see what's actively going on with auto tasks? Needs: current state (what's running, what's queued, what's stalled), recent history (what completed, what failed), and controls (re-prioritize, trigger retry, inspect). Execution state written back to task bodies is the data foundation; the question is the interface on top.
